# 校准方法论 — 自我进化参数优化指南

## 校准闭环

```
预测 → 赛后复盘 → 多维评分 → 偏差分析 → 参数调整 → 下次预测
  ↑                                                          ↓
  └────────────────── 进化引擎 ──────────────────────────────┘
```

## 校准维度

### 1. Elo K值
- **触发条件**：每3场复盘后自动评估
- **调整逻辑**：准确率<40% → K↓; 准确率>60% → K↑
- **范围**：[16, 64]
- **世界杯淘汰赛**：建议K=48（比常规K=32更高，反映淘汰赛信息量更大）

### 2. Elo vs 泊松权重
- **触发条件**：每3场复盘后，对比Elo独立准确率与整体准确率
- **调整逻辑**：Elo准确率 > 整体+10% → ELO_WEIGHT↑; Elo准确率 < 整体-10% → ELO_WEIGHT↓
- **范围**：[0.15, 0.50]

### 3. 首轮修正因子
- **OPENER_DRAW_BOOST**（首轮平局加成）
  - 基于实际首轮平局率动态调整
  - 历史基准：2026世界杯首轮4场100%平局 → 加成0.15
  - 调整幅度：±0.03/轮
- **FAV_GOAL_DISCOUNT**（强队进球打折）
  - 基于首轮xG偏差动态调整
  - 历史基准：0.50
- **UNDERDOG_GOAL_BOOST**（弱队进球加成）
  - 历史基准：0.40

### 4. 攻防系数
- 每场比赛后使用指数滑动平均更新（alpha=0.3）
- 新球队默认使用联赛平均值LEAGUE_AVG_GOALS=1.35

## 复盘评分体系（100分制）

| 维度 | 满分 | 评分逻辑 |
|------|------|----------|
| 方向 | 30 | 方向正确30，平局/一方正确10，全错0 |
| 比分 | 30 | 精确匹配按概率×3，相近总分20上限，否则5 |
| xG | 20 | 误差<0.5球20分，<1球15分，<2球10分，否则5分 |
| 赔率 | 20 | 预测低概率事件正确20分，中等15分，高概率10分 |

## 进化数据存储

- `logs/review_log.jsonl`：每场复盘记录（JSONL格式，可追加）
- `logs/evolution_state.json`：进化引擎状态（参数历史、准确率时间线）
- `logs/profit_track.jsonl`：收益追踪记录
- `logs/profit_state.json`：收益状态

## 安全边界

- 参数调整幅度：单次不超过 ±20%
- 至少3场复盘数据才能触发自动进化
- 所有参数变更记录到evolution_state.json的param_history
- 支持手动回滚到任意历史参数快照
