一个能**自我进化**的世界杯足球预测系统，整合量化模型、贝叶斯分析、赛后复盘和参数自校准，持续提高预测准确率和押注收益。

### 核心规律（v2.1 验证，v4.0 固化为可进化参数）

> **强队表现按轮次变化巨大，不能用单一系数。**
> R1保守(0.65x) → R2反弹(1.10x+动量加成) → R3常态(1.00x) → KO防守(0.95x)
> 模型必须按轮次自适应，否则会在R1高估、R2低估之间反复出错。

### v4.2 关键进化（2026-06-23）

> **纯泊松比分选择至今0%命中。市场人气(participantCount)是有效的交叉验证信号。**
> 当top2泊松比分概率差<3%时，优先选市场共识更高的那个。
> 总进球：泊松P(k)与市场共识k分歧>3倍时，跟随市场。
> R1→R2攻击动量：首轮进3+球的球队，R2 xG自动+15%（挪威4-1→R2攻击力被低估）

### v4.4 关键进化（2026-07-01）

> **KO赛三个系统性缺陷：东道主效应、球星加成、让球盘偏差。**
> 东道主效应：2026联合东道主(墨/加/美)KO赛xG+15%，对手xG-10%。墨西哥2-0厄瓜多尔验证。
> 球星加成：Mbappé/Haaland/Bellingham等超级巨星所在球队KO赛xG+10%。法国3-0瑞典(Mbappé双响)验证。
> 让球风险标记：R32让球方向0/6命中，让球盘与FT方向冲突时标注"让球高风险"。
> 策略优先级：HT/FT Top1(67%) > 总进球Top2(67%) > FT方向(33%) > 让球方向(0%) > 比分(0%)。

## 工作流程

### 1. 赛前预测

```
用户输入比赛信息
    ↓
加载预测引擎 (scripts/prediction_engine.py)
    ↓
├─ 判断比赛轮次 (R1/R2/R3/KO) → 选择对应系数
├─ Elo评级对比 (data/elo_ratings.json)
├─ 泊松xG计算 (data/team_stats.json)
├─ 修正因子应用 (data/corrections.json)
├─ 贝叶斯战术语境修正
│   ├─ 控球陷阱检测
│   ├─ 风格克制分析
│   └─ 联赛/赛制特性
├─ 赔率获取与价值检测 (scripts/odds_fetcher.py)
└─ Kelly仓位计算
    ↓
输出: 预测报告 + 剧本 + 风险提示
    ↓
记录预测到 logs/prediction_log.md
```

### 2. 赛后复盘

```
用户提供实际比分 或 系统联网搜索
    ↓
加载复盘引擎 (scripts/post_match_review.py)
    ↓
├─ 方向对比: 预测方向 vs 实际结果
├─ 比分对比: 预测比分 vs 实际比分
├─ xG误差: 预测进球 vs 实际进球
├─ 赔率建议评分
└─ 综合评分 (100分制)
    ↓
├─ 更新Elo (预测引擎 update_elo)
├─ 更新球队攻防统计 (滑动平均)
├─ 保存复盘记录 (logs/review_log.jsonl)
└─ 生成进化建议
```

### 3. 自我进化（v4.0 增强）

```
累积≥3场复盘数据后自动触发
    ↓
加载进化引擎 (scripts/evolution_engine.py)
    ↓
├─ 按轮次分组复盘数据 (R1/R2/R3/KO)
├─ 每轮独立校准:
│   ├─ 方向准确率 → 调整 fav_discount
│   ├─ xG偏差 → 调整 underdog_boost
│   └─ 平局率 → 调整 draw_boost
├─ 传统参数校准:
│   ├─ Elo独立准确率 vs 整体 → 调整 ELO_WEIGHT
│   └─ 整体准确率 → 调整 K_FACTOR
├─ v4.0 过拟合检测:
│   ├─ 相邻轮次反向检测 (R1↓+R2↑跨度>0.45)
│   ├─ 系数变化幅度检查 (>0.20警告)
│   └─ 数据量不足警告 (<3场低置信度)
├─ v4.0 回测验证:
│   ├─ 总偏差、方向准确率、平均xG误差
│   └─ 按轮次分组统计
└─ 注入系数到 prediction_engine
    ↓
├─ 记录进化事件到 evolution_state.json
├─ 更新准确率时间线
└─ 生成进化报告 (含轮次系数表+过拟合警告)
```

### 4. 收益追踪

```
赛后结算 → 收益追踪器 (scripts/profit_tracker.py)
    ↓
├─ 模拟投注 (Kelly半仓/固定比例/价值投注)
├─ 计算单场盈亏
├─ 更新累计收益
├─ 追踪最大回撤
└─ 按策略分组统计ROI
    ↓
策略优化建议
```

## 数据文件

- `data/elo_ratings.json`：48支世界杯参赛队Elo评级（赛后自动更新）
- `data/team_stats.json`：各队攻防数据（场均进球/失球，赛后滑动平均更新）
- `data/corrections.json`：修正因子参数库（含首轮修正、爆冷修正等）
- `data/world_cup_schedule.json`：世界杯完整赛程

## 日志文件

- `logs/prediction_log.md`：赛前预测记录
- `logs/review_log.jsonl`：赛后复盘记录（JSONL格式，每行一场）
- `logs/evolution_state.json`：进化引擎状态（参数历史+准确率时间线+轮次系数+过拟合警告）
- `logs/profit_track.jsonl`：收益追踪记录
- `logs/profit_state.json`：收益状态（当前资金+ROI）

## 使用指南

### 单场比赛预测

1. 解析用户输入，识别球队名称和比赛信息
2. 判断比赛轮次（从 `data/world_cup_schedule.json` 读取，不能凭直觉判断）
3. 联网搜索最新伤停、首发、赔率信息（可选）
4. 加载引擎，调用 `engine.predict(team_a, team_b, tournament_round='group_stage_round1')` 
5. **v4.2增强**: 对于有市场数据的场景，使用 `engine.predict_with_consensus()` 融合泊松与市场人气:
   ```python
   engine.set_r1_scoring({'挪威': 4, '英格兰': 4, ...})  # 先注入首轮进球
   pred = engine.predict_with_consensus(
       '挪威', '塞内加尔',
       market_scores={"1:0": 0, "2:0": 25, ...},
       market_totals={"0": 0, "1": 0, "2": 23, "3": 7, ...},
       tournament_round='group_stage_round2'
   )
   ```
6. 应用贝叶斯战术语境修正（控球陷阱+风格克制+联赛特性）
7. 生成P0→P1概率更新和剧本预测
8. 如需赔率分析，调用 `scripts/odds_fetcher.py`
9. 输出完整预测报告，附风险提示

### 赛后复盘

1. 联网搜索比赛结果：`"{team_a} {team_b} 比分 比赛结果"`
2. 加载复盘引擎，调用 `reviewer.review_match()`
3. 自动更新Elo和球队统计
4. 输出复盘评分和进化建议
5. 触发进化引擎检查是否需要参数校准

### 自我进化

1. 累积3场以上复盘数据后，调用 `evolution_engine.evolve()`
2. v4.0引擎自动按轮次分组校准、执行过拟合检测、运行回测验证
3. 生成进化报告，展示参数变更、轮次系数表、过拟合警告
4. 所有变更记录到 `logs/evolution_state.json`
5. 手动回测：`python scripts/evolution_engine.py --backtest`

### 收益追踪

1. 每次预测后调用 `profit_tracker.simulate_bet()` 模拟投注
2. 赛后根据实际比分自动结算
3. 调用 `profit_tracker.get_profit_report()` 查看累计收益
4. 调用 `profit_tracker.optimize_strategy()` 获取策略优化建议
