# 国际足联世界杯（FIFA World Cup）足球预测计算器

一个能**自我进化**的世界杯足球预测系统，整合量化模型、贝叶斯分析、赛后复盘和参数自校准，持续提高预测准确率和押注收益。

## 核心能力

### 🎯 预测能力（整合自两个源技能）

| 能力 | 来源 | 说明 |
|------|------|------|
| **单场预测** | 量化分析 | Elo评级+泊松分布+蒙特卡洛模拟 → 胜平负概率+最可能比分 |
| **爆冷分析** | 量化分析 | 三层判据（风格克制+状态变量+赛制红利）→ 爆冷概率+等级判定 |
| **晋级概率** | 量化分析 | 蒙特卡洛10万次模拟推演各阶段晋级路径 |
| **赔率差检测** | 量化分析 | 模型概率 vs 博彩市场隐含概率，≥3%偏差标记价值信号+Kelly仓位 |
| **贝叶斯先验→后验** | 贝叶斯分析 | P0(先验) → 战术语境修正 → 联赛特性修正 → 似然更新 → P1(后验) |
| **战术语境分析** | 贝叶斯分析 | 控球陷阱检测 + 风格克制分析 + 联赛/赛制特性修正 |
| **剧本预测** | 贝叶斯分析 | 基于战术机制生成2-4套不同比赛剧本 |
| **v2.1轮次自适应** | 量化分析 | 根据tournament_round自动应用4套不同系数：R1保守(×0.5) → R2反弹(×1.10) → R3常态(×1.00) → KO防守(×0.95) |
| **v3.1实时数据** | 量化分析 | BSD API集成：赛前1h获取首发阵容+阵型，赛前获取伤停报告（受伤/停赛/存疑），实时赔率多源对比 |
| **v3.2爆冷进化** | 量化分析 | 三代爆冷分析：三层判据(风格克制+状态变量+赛制红利)+match_context参数+Tier分级(1/2/3) |

### 🆕 进化能力（v3.0 新增）

| 能力 | 说明 |
|------|------|
| **赛后复盘** | 联网搜索真实比赛结果 → 与预测对比 → 多维评分(方向/比分/xG/赔率) |
| **Elo动态更新** | 赛后自动更新Elo评级，K值自适应（小组赛K=32/淘汰赛K=48） |
| **参数自校准** | 基于复盘数据自动优化：Elo权重、平局加成、进球修正、K因子 |
| **准确率追踪** | 追踪整体/近期方向准确率、趋势检测(improving/stable/declining) |
| **收益追踪** | 虚拟投注模拟 → ROI计算 → 策略对比(Kelly半仓/固定比例/价值投注) |
| **进化报告** | 生成参数变更历史、准确率时间线、策略优化建议 |

## 工作流程

### 1. 赛前预测

```
用户输入比赛信息
    ↓
加载预测引擎 (scripts/prediction_engine.py)
    ↓
├─ Elo评级对比 (data/elo_ratings.json)
├─ 泊松xG计算 (data/team_stats.json)
├─ 修正因子应用 (data/corrections.json)
├─ 贝叶斯战术语境修正
│   ├─ 控球陷阱检测
│   ├─ 风格克制分析
│   └─ 联赛/赛制特性
├─ 赔率获取与价值检测 (scripts/odds_fetcher.py)
└─ Kelly仓位计算
    ↓
输出: 预测报告 + 剧本 + 风险提示
    ↓
记录预测到 logs/prediction_log.md
```

### 2. 赛后复盘（🆕）

```
用户提供实际比分 或 系统联网搜索
    ↓
加载复盘引擎 (scripts/post_match_review.py)
    ↓
├─ 方向对比: 预测方向 vs 实际结果
├─ 比分对比: 预测比分 vs 实际比分
├─ xG误差: 预测进球 vs 实际进球
├─ 赔率建议评分
└─ 综合评分 (100分制)
    ↓
├─ 更新Elo (预测引擎 update_elo)
├─ 更新球队攻防统计 (滑动平均)
├─ 保存复盘记录 (logs/review_log.jsonl)
└─ 生成进化建议
```

### 3. 自我进化（🆕）

```
累积≥3场复盘数据后自动触发
    ↓
加载进化引擎 (scripts/evolution_engine.py)
    ↓
├─ 分析Elo独立准确率 vs 整体准确率 → 调整ELO_WEIGHT
├─ 分析首轮平局实际率 → 调整OPENER_DRAW_BOOST
├─ 分析首轮xG偏差 → 调整FAV_GOAL_DISCOUNT / UNDERDOG_GOAL_BOOST
├─ 分析整体准确率 → 调整K_FACTOR
└─ 应用新参数到预测引擎
    ↓
├─ 记录进化事件到 evolution_state.json
├─ 更新准确率时间线
└─ 生成进化报告
```

### 4. 收益追踪（🆕）

```
赛后结算 → 收益追踪器 (scripts/profit_tracker.py)
    ↓
├─ 模拟投注 (Kelly半仓/固定比例/价值投注)
├─ 计算单场盈亏
├─ 更新累计收益
├─ 追踪最大回撤
└─ 按策略分组统计ROI
    ↓
策略优化建议
```

## 数据文件

- `data/elo_ratings.json`：48支世界杯参赛队Elo评级（赛后自动更新）
- `data/team_stats.json`：各队攻防数据（场均进球/失球，赛后滑动平均更新）
- `data/corrections.json`：修正因子参数库（含首轮修正、爆冷修正等）
- `data/world_cup_schedule.json`：世界杯完整赛程

## 日志文件（🆕）

- `logs/prediction_log.md`：赛前预测记录
- `logs/review_log.jsonl`：赛后复盘记录（JSONL格式，每行一场）
- `logs/evolution_state.json`：进化引擎状态（参数历史+准确率时间线）
- `logs/profit_track.jsonl`：收益追踪记录
- `logs/profit_state.json`：收益状态（当前资金+ROI）

## 使用指南

### 单场比赛预测

1. 解析用户输入，识别球队名称和比赛信息
2. 判断比赛轮次（小组赛首轮/末轮/淘汰赛等）
3. 联网搜索最新伤停、首发、赔率信息（可选）
4. 加载引擎，调用 `engine.predict()` 计算
5. 应用贝叶斯战术语境修正（控球陷阱+风格克制+联赛特性）
6. 生成P0→P1概率更新和剧本预测
7. 如需赔率分析，调用 `scripts/odds_fetcher.py`
8. 输出完整预测报告，附风险提示

### 赛后复盘

1. 联网搜索比赛结果：`"{team_a} {team_b} 比分 比赛结果"`
2. 加载复盘引擎，调用 `reviewer.review_match()`
3. 自动更新Elo和球队统计
4. 输出复盘评分和进化建议
5. 触发进化引擎检查是否需要参数校准

### 自我进化

1. 累积3场以上复盘数据后，调用 `evolution_engine.evolve()`
2. 引擎自动分析复盘数据，校准核心参数
3. 生成进化报告，展示参数变更和准确率趋势
4. 所有变更记录到 `logs/evolution_state.json`

### 收益追踪

1. 每次预测后调用 `profit_tracker.simulate_bet()` 模拟投注
2. 赛后根据实际比分自动结算
3. 调用 `profit_tracker.get_profit_report()` 查看累计收益
4. 调用 `profit_tracker.optimize_strategy()` 获取策略优化建议

## 红线

- 绝不直接建议投注，提供分析框架
- 绝不伪造数据，所有计算通过代码执行
- 绝不隐瞒偏差，模型vs市场大偏差时标注不确定性
- 每次分析结尾必须附风险提示
- 不预测点球大战结果
- 联网搜索必须标注数据来源和时间戳
- 所有数据缺失必须标记为 UNAVAILABLE 并说明原因
